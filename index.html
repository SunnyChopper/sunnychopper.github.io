HTML5

HTML is the backbone of any website in my opinion and so I’ve spent over 6 years playing around and mastering how HTML5 works to create websites that operate great and are SEO-friendly.

CSS3

While learning HTML5, I realized that if I wanted to create sites that looked great, I would have to learn about CSS3. As mobile took over the market, things like media queries became my focus.

Javascript

Once I was used to making great looking websites, the next obvious step for me was to learn JavaScript to make my websites more interactive in order to give a better user experience to visitors.

jQuery

After taking a deep-dive into JavaScript, I discovered the jQuery library that made DOM manipulation so much easier. I was able to quickly pick up jQuery to make my websites more interactive.

PHP

While creating iOS applications in high school, I quickly realized that if I wanted to connect my apps with any public API, I would have to create a backend server to interface with these APIs.

Python

At first, I used Python for data structures class. However, after realizing how sleek the language was and the community support, I quickly learned Python and frameworks such as Django.

Java

Although Java is not my go-to language for backend development, I have experience from coursework. For one of my classes, I was required to develop an Android app using Java.

C++

My C++ experience primarily comes from working on coursework and hobby projects. The coolest thing I’ve built so far with C++ was a mock banking system for a class project.

Objective-C

The first few mobile applications that I created in high school were built in Obj-C. Since then, I have focused my time on learning more about Swift as Apple has shifted their focus to that.

Swift

After Swift was released in 2014, I started to move away from Obj-C and into Swift. It did take me time to transition, however, I was able to learn Swift and managed to create a few hobby iOS apps.

SQL

SQL was extremely easy for me to pick up as the query language seemed very intuitive to me. Since first learning, I’ve taken a deeper dive into concepts such as JOINs, sub-queries and more.

R

For my AP Statistics class, I wanted to go the extra mile and learn about R as I thought it would help me learn statistics faster. R’s language was easy to me and would love to return soon.

SciKit Learn

My curiosity for artificial intelligence and machine learning lead me to learn about the scikit-learn library for Python. I was able to get hands-on experience with it while taking an AI course.

TensorFlow

While in my AI course, I was exposed to TensorFlow and how it can be used to build complex deep neural networks. I plan to learn more about it after I polish my math skills for AI.

Laravel

The first web development framework that I got my hands dirty with was Laravel. I used Laravel for many of my hobby projects and managed to use it for one of my larger freelance projects.

Web2Py

While working at Quicket Solutions as a software engineering intern, I was exposed to the Web2Py framework. Since I already experience with Laravel, I was able to pick it up within a day.

Django

After I realized that I would like to move away from PHP and Laravel and towards Python for artificial intelligence, I quickly picked up Django to replace Laravel.

React

While learning Laravel, I was reading a lot about single-page applications and how client-side rendering was the new hot thing. So I learned React to supplement my Laravel knowledge.

React Native

After learning React, I realized that learning React Native wouldn’t be too much harder. The additional benefit of only having to write your code once and deploy to both iOS and Android was attractive.

Scrum

I’ve used Scrum while working at Quicket Solutions and also for managing projects for school in the event I was the project leader. Familiar with the process and read Scrum by Jeff Sutherland.

GitHub

Prior to working at a real developer job, I would store code on my local machine, however, since joining Quicket, I’ve learned Git and how it’s used for version control and deploying production code.

Jira

While working at Quicket Solutions, I was exposed to Jira and how it was used in a real-world setting. At first, it was hard to pick-up but by the end, I found it to be a very useful tool.

Docker

I am just now starting to learn about Docker and Kubernetes. I’ve created some “Hello World” docker images, however, have yet to use them in a hobby project or in a real world job.

Amazon AWS

Have experience with deploying and setting up EC2 instances. Also have experience setting up RDS instances to connect with the EC2 instance in the case that I am using MySQL for development.
